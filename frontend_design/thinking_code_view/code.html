Can you write a robust TypeScript interface and a custom hook for managing complex form state with validation?

To handle complex form states effectively in TypeScript, I recommend a compositional pattern that decouples validation logic from the component lifecycle.

Key Principles

  • Strict Type Inference
  • Memoized Change Handlers
  • Asynchronous Validation

Architecture

The following implementation uses a generic structure to support any data shape while maintaining type safety.

useForm.ts 
interface FormState<T> {
  // Core state definitions
  values: T;
  errors: Partial<Record<keyof T, string>>;
  touched: Partial<Record<keyof T, boolean>>;
  isSubmitting: boolean;
}

export const useForm = <T>(initialValues: T) => {
  const [state, setState] = useState<FormState<T>>({
    values: initialValues,
    errors: {},
    touched: {},
    isSubmitting: false
  });

  // Liquid change handler with generic inferring
  const handleChange = (key: keyof T, value: any) => {
    setState(prev => ({
      ...prev,
      values: { ...prev.values, [key]: value }
    }));
  };

  return { ...state, handleChange };
};

Note: This pattern works seamlessly with library-based validation like Zod or Yup by passing a schema validator to the hook configuration.
